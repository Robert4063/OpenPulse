## 系统简介

OpenPulse 是一个开源项目数据分析平台，收录了 GitHub 上 **Top 300** 热门开源项目的历史数据，包括：

- ⭐ Star（收藏）趋势
- 🍴 Fork（分叉）趋势  
- 📋 Issue（问题）趋势
- 💬 Comment（评论）趋势
- 👥 贡献者分布
- 🏥 项目健康度评分

**数据时间范围**：2022年3月 - 2023年3月

## 首页功能

### 1. 项目搜索

在首页顶部的搜索框中，您可以：

- **关键词搜索**：输入项目名称或关键词（如 "react"、"tensorflow"）
- **Star 数量筛选**：设置最小和最大 Star 数量来筛选项目

### 2. 项目列表

搜索结果以卡片形式展示，每个项目卡片显示：

- 项目名称（owner/repo 格式）
- Star 数量
- Fork 数量
- 健康度等级（A/B/C/D/E）

### 3. 进入项目详情

点击任意项目卡片，即可进入该项目的详情页面。

## 项目详情页

项目详情页包含以下几个主要模块：

### 1. 项目基本信息

页面顶部显示项目的：
- 名称
- 总 Star 数
- 总 Fork 数
- 健康度等级

### 2. 趋势图表

提供四种趋势图表，可通过标签切换：

| 图表类型 | 说明 |
|---------|------|
| ⭐ Star 趋势 | 每日新增 Star 数量变化 |
| 🍴 Fork 趋势 | 每日新增 Fork 数量变化 |
| 📋 Issue 趋势 | 每日新增 Issue 数量变化 |
| 💬 Comment 趋势 | 每日评论数量变化 |

**图表交互**：
- 鼠标悬停可查看具体日期的数值
- 支持缩放和平移（在某些图表上）

### 3. 贡献者分布

以饼状图形式展示项目的 Top 贡献者及其贡献占比。

### 4. 健康度评分面板

详细展示项目的健康度评分，包括：
- 综合得分（0-100）
- 健康等级（A/B/C/D/E）
- 四维雷达图
- 各维度详细数据

## 健康度评分解读

OpenPulse 使用 **PHAM v2.0**（Project Health Assessment Model）模型评估项目健康度。

### 等级说明

| 等级 | 分数范围 | 含义 | 颜色 |
|-----|---------|------|------|
| A | 80-100 | 优秀 | 🟢 绿色 |
| B | 60-79 | 良好 | 🔵 蓝色 |
| C | 40-59 | 一般 | 🟡 黄色 |
| D | 20-39 | 较差 | 🟠 橙色 |
| E | 0-19 | 危险 | 🔴 红色 |

### 四大评估维度

#### 1. 关注度增长（权重 20%）

评估项目的 Star 和 Fork 增长趋势。

- **Star 增长得分**：当前月 Star 数与过去3个月平均值的比较
- **Fork 增长得分**：当前月 Fork 数与过去3个月平均值的比较

**解读**：
- 得分 > 50：增长势头良好，关注度上升
- 得分 = 50：保持稳定
- 得分 < 50：关注度下降

#### 2. 开发活跃度（权重 40%）

评估项目的代码提交活跃程度。

- **Commit 趋势得分**：最近一周 Commit 与月均的比较
- **OpenDigger 活跃度**：基于多种活动类型的综合评分

**解读**：
- 高分表示项目持续有活跃开发
- 低分可能意味着项目维护频率降低

#### 3. 社区贡献度（权重 20%）

评估社区参与度和 PR（Pull Request）活跃程度。

- **周均 PR**：最近一周平均每天的 PR 数量
- **月均 PR**：本月平均每天的 PR 数量
- **PR 趋势比**：周均与月均的比值

**解读**：
- 高分表示社区积极贡献代码
- 趋势比 > 1 表示贡献热度上升

#### 4. 代码健康度（权重 20%）

评估代码变动量和维护状态。

- **代码添加**：PR 中新增的代码行数
- **代码删除**：PR 中删除的代码行数
- **总变动量**：代码总体变动规模

**解读**：
- 适度的代码变动表示项目在积极维护
- 完全没有变动可能意味着项目停滞

### 查看详细数据

在健康度评分面板中，点击任意维度条可展开查看：
- 饼状图：子指标占比
- 详细数值：原始统计数据

### 相似项目推荐

点击健康度评分面板的标题区域，可打开相似项目推荐弹窗，查找与当前项目健康度相近的其他项目。

## 常见问题

### Q1: 为什么某些项目没有数据？

A: 本系统收录的是 2022年3月至2023年3月期间的 GitHub Top 300 项目数据。如果项目在该时期之外或不在收录范围内，则无法显示数据。

### Q2: 数据更新频率是多少？

A: 当前数据为历史快照数据（2022.03 - 2023.03），不会实时更新。

### Q3: 健康度评分的参考基准是什么？

A: 评分基准时间为 2023年3月，所有趋势计算都以该时间点为基准。

### Q4: 为什么有些项目的某些指标显示为0？

A: 可能的原因：
- 该项目在统计期间内该指标确实为0
- 部分数据可能未完整收录

### Q5: 如何对比多个项目？

A: 您可以：
1. 分别打开多个项目详情页进行对比
2. 使用相似项目推荐功能查找相近项目
